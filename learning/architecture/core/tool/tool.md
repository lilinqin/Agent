# 工具系统

> 适用：技术面试、工具设计

---

## 工具本质

**工具是 Agent 与外部世界交互的桥梁**。

核心问题：如何让 LLM 知道什么时候用什么工具？

---

## 工具定义

```json
{
  "name": "search",
  "description": "当你需要查询实时信息时使用",  // 触发场景
  "parameters": { ... }
}
```

**面试话术**：
> "工具描述的重点是'什么时候用'，不是'怎么用'。"

---

## 两种调用方式

| | Function Calling | ReAct |
|---|---|---|
| 原理 | 模型直接输出结构化参数 | 模型输出自然语言格式 |
| 优点 | 格式可靠，无需解析 | 通用性强，任何模型都能用 |
| 缺点 | 需要模型支持 function calling | 需要正则解析，可能格式错误 |

**生产环境推荐 Function Calling**，原生支持更稳定。

---

## MCP 协议

Anthropic 提出的标准化工具调用协议：
- 标准化：统一工具定义和调用格式
- 可发现：工具可动态注册和发现
- 安全：明确的权限边界

---

## 工程要点

1. **描述即提示词**：描述触发场景，不是实现细节
2. **错误处理**：临时错误重试，永久错误返回明确信息
3. **重试机制**：临时错误自动重试 2-3 次
4. **调用限制**：频率限制 + 总量限制 + 成本监控

---

## 面试高频

**Q: ReAct 和 Function Calling 怎么选？**
> "Function Calling 格式可靠但需要模型支持；ReAct 更通用但需要解析。生产环境首选 Function Calling。"

**Q: 工具调用失败怎么办？**
> "分情况：临时错误（网络波动）重试；参数错误返回具体错误让 LLM 调整；权限错误触发人工介入。"
