# 记忆系统

> 适用场景：技术面试 / 长对话系统设计 / RAG 规划

---

## 核心概念（一句话定义）

记忆系统解决的是"Agent 在时间维度上的信息持久化"问题——**LLM 本身无状态，记忆系统让它能"记住"过去、检索相关知识、跨会话持续进化**。

---

## 为什么需要记忆

LLM 有两个根本性局限：

1. **对话状态遗忘**：LLM 设计上无状态，每次 API 调用独立计算，不自动记住上一轮内容。多轮对话后，早期关键信息消失，个性化缺失，前后矛盾
2. **内置知识局限**：训练截止日期后无新知识；专业领域知识深度不足；回答可能产生幻觉

这就是为什么做复杂 Agent 必须认真设计记忆系统——**不是"可选功能"，而是长程任务的必要前提**。

---

## 记忆分层架构

### 功能层次（三层）

| 层次 | 存储位置 | 生命周期 | 作用 |
|------|----------|----------|------|
| **工作记忆** | context window | 单次任务 | 当前任务执行的即时上下文 |
| **会话记忆** | 变量 / DB | 会话期间 | 多轮对话历史，保持对话连贯性 |
| **长期记忆** | 向量/图数据库 | 持久化 | 跨会话知识，用户偏好，领域知识 |

### 记忆类型（四种，按认知科学分类）

借鉴认知心理学，可以将长期记忆进一步细分：

| 类型 | 类比人类记忆 | 存储内容 | 存储后端 | 特征 |
|------|------------|----------|----------|------|
| **工作记忆 (Working)** | 工作记忆 | 当前对话上下文 | 内存/缓存 | TTL 管理，默认上限 50 条 |
| **情景记忆 (Episodic)** | 情景记忆 | 具体交互事件和经历 | 向量数据库 | 时间序列，支持回顾检索 |
| **语义记忆 (Semantic)** | 语义记忆 | 抽象知识、概念、规则 | 向量 + 知识图谱 | 持久性高，支持关系推理 |
| **感知记忆 (Perceptual)** | 感觉记忆 | 图像、音频等多模态数据 | 多模态存储 | 跨模态检索，动态生命周期 |

**两种分法的关系**：功能三层是工程视角（如何管理），认知四类是内容视角（存什么）。实际系统两者结合——工作记忆对应第一层，情景+语义+感知对应长期记忆层的不同子类型。

### 记忆生命周期

```
感知输入 → 编码 (Encoding)
         → 存储 (Storage)  →  情景/语义/感知记忆
         → 检索 (Retrieval) ← 相关性搜索
         → 整合 (Consolidation): 工作记忆 → 长期记忆
         → 遗忘 (Forgetting): 基于重要性/时间/容量
```

遗忘不是错误，而是必要的——它防止低质量记忆占据有限的注意力预算。

## 记忆管理策略

### 截断

只保留最近 N 轮或 M tokens。简单但可能丢失重要信息。适合短任务。

### 摘要（Compaction）

每轮结束后用 LLM 生成摘要，压缩历史。优点是高压缩率，缺点是每次都要调用 LLM。

最佳实践（来自 Claude Code 工程实践）：
- 触发条件：L3（动态会话层）超过 context window 的 80%
- 生成固定格式摘要：原始目标 + 技术上下文 + 已完成里程碑 + 关键决策 + 文件状态
- 摘要本身不参与再次压缩（防止层层失真）

### RAG（检索增强）

把历史和知识存向量库，按相关性检索"召回"。适合长程任务、大规模知识库。

**RAG 的演化三阶段**：

| 阶段 | 时间 | 检索方式 | 生成优化 |
|------|------|---------|---------|
| **朴素 RAG (Naive RAG)** | 2020-2021 | TF-IDF / BM25 关键词匹配 | 直接拼接检索结果 |
| **高级 RAG (Advanced RAG)** | 2022-2023 | 稠密嵌入语义检索 | 查询重写、文档分块、重排序 |
| **模块化 RAG (Modular RAG)** | 2023至今 | 混合检索、多查询扩展、HyDE | 思维链推理、自我反思修正 |

**高级检索策略**：
- **MQE（多查询扩展）**：将用户问题扩展为多个角度的查询，提升召回率
- **HyDE（假设文档嵌入）**：先让 LLM 生成假设答案，用假设答案的嵌入去检索，更接近目标文档的语义空间

## 工程实践

### 记忆 vs RAG 的选择

| 场景 | 优先选择 |
|------|---------|
| 需要记住用户偏好、历史交互 | 记忆系统（MemoryTool） |
| 需要查询外部知识库、文档 | RAG（RAGTool） |
| 需要关系推理、多跳查询 | 语义记忆（Neo4j 图数据库） |
| 多模态输入（图像、音频） | 感知记忆 |

### 上下文利用率的甜蜜区间

实践发现：context window 超过某个阈值后，模型性能下降（"上下文腐蚀"）。顶级工程师通常将利用率控制在 **60% 以下**，多余的用压缩/遗忘/子代理来处理。

### 整合（Consolidation）触发条件

工作记忆 → 长期记忆的整合时机：
- 重要性评分超过阈值（用户明确表示"记住这个"）
- 访问频次高的信息
- 任务完成时的关键发现

## 当前发展现状

**技术成熟度**：
- 分层架构已成为共识，RAG 技术成熟
- 情景 + 语义 + 感知的细粒度分类是 2024-2025 年的最新趋势
- 摘要/压缩技术仍在快速演进

**主要矛盾**：
- **成本 vs 效果**：摘要需要 LLM 调用，向量存储需要 embedding；都有成本
- **检索质量**：向量检索依赖 embedding 质量，长文档的分块策略影响大
- **一致性**：跨层记忆的一致性维护（工作记忆中的内容 vs 长期记忆中的知识冲突）

**行业趋势**：
- Claude Code 的 compaction 机制验证了"摘要 + 重启 context"的有效性
- 大量企业开始用 Qdrant（向量）+ Neo4j（图）组合实现语义记忆
- "记忆即数据飞轮"：harness 捕获的交互轨迹回流训练，形成能力增强闭环

## 未来展望

**短期（1-2年）**：
- 记忆分层会更精细（4类型已成为框架标准）
- Episodic Memory 的自动整合为 Semantic Memory 会成为亮点特性
- 个性化记忆成为 AI 助手差异化的关键点

**长期（3-5年）**：
- 可能出现"记忆原生"的 LLM（训练时内建记忆机制）
- 多模态记忆（图像、语音）成为标配
- 隐私保护的联邦记忆架构

## 深度理解

**Q: 上下文窗口管理有哪些策略？**

> 1. **截断**——只保留最近 N 轮，简单但丢失信息；2. **摘要**——用 LLM 压缩历史，高压缩率但有成本；3. **RAG**——向量检索召回相关历史，最灵活但复杂度高。根据任务长度和质量要求选择，长程任务建议三者结合。

**Q: Agent 记忆和普通对话历史有什么区别？**

> 普通对话只需存消息列表。Agent 记忆要区分四种类型（工作/情景/语义/感知），每种有不同的存储后端、生命周期和检索策略；还要处理记忆整合（短期→长期）、遗忘（容量/时间/重要性策略）等问题，是一套完整的认知管理系统。

**Q: 为什么语义记忆要用知识图谱（Neo4j）而不是只用向量数据库？**

> 向量数据库擅长"语义相似性检索"，但对"关系推理"能力有限。语义记忆存的是抽象概念和它们之间的关系（A is_a B, A depends_on C），这种结构天然适合图数据库。多跳查询（"找到所有依赖 X 的服务的所有直接下游"）只有图数据库能高效完成，向量检索做不到。

## 一句话总结

记忆系统核心是**分层管理**：工作记忆保执行、情景记忆保经历、语义记忆保知识、感知记忆保多模态——通过截断/摘要/检索控制上下文长度，通过整合/遗忘维护记忆质量。
