# 大语言模型基础

> 适用场景：技术面试、内部分享、概念梳理

---

## 从"下一个词"说起

语言模型做的事情听起来特别简单：**预测下一个最可能出现的词**。

你打完"今天天气真"，模型会猜你接下来想说"好"——因为在它见过的大量文本里，"天气真"后面跟"好"的概率最高。

这个看似简单的任务，恰恰是所有大语言模型的根基。不管是 ChatGPT 还是 Claude，本质都是在玩一个超高维度的"文字接龙"游戏。

---

## 语言模型的进化之路

### N-gram：数数的智慧

最早的语言模型叫 **N-gram**，核心思想很朴素：一个词出现概率，只和它前面 n-1 个词有关。

- Bigram (N=2)：只看前面 1 个词
- Trigram (N=3)：只看前面 2 个词

这叫**马尔可夫假设**——别想太远，眼前这几个词够用了。

但 N-gram 有两个致命伤：
- **数据稀疏**：没见过的词组合，概率直接标 0
- **不懂语义**：它不知道"agent"和"robot"其实差不多

### 词嵌入：把词变成向量

2003 年，Bengio 提出了一个革命性的想法：**用连续的向量来表示词**。

这就是**词嵌入 (Word Embedding)**。在这个向量空间里：
- 语义相近的词，向量也相近
- 还能做向量运算：`King - Man + Woman ≈ Queen`

但问题是——上下文窗口还是固定的。看"今天天气真好"这几个字，模型只能看到前面几个词，再早的就忘了。

### RNN/LSTM：引入记忆

**循环神经网络 (RNN)** 解决了这个问题。它有个隐藏状态，像个"记忆细胞"，能把信息从前面传递到后面。

但 RNN 有个老毛病：**梯度消失**。序列一长，早期的信息就传不过来了。

**LSTM** 通过门控机制（遗忘门、输入门、输出门）来控制信息的存留，总算能记住更长序列了。

但它必须**顺序计算**，无法并行——这意味着训练太慢，模型做不大。

## Transformer：注意力就是一切

2017 年，谷歌发表了那篇著名的 "Attention is All You Need"。

Transformer 彻底抛弃了循环结构，用**自注意力机制**来处理序列。

### 注意力机制到底在做什么

想象你读这句话：
> "The agent uses *its* tool to solve the problem."

读到 "its" 时，你的大脑会自然地把注意力放在 "agent" 上，而不是其他无关的词。

**自注意力就是对这个过程的数学建模**：
- 每个词都会"询问"其他所有词："谁和我有关？"
- 通过计算相关性分数，决定每个词应该从其他词那里"吸收"多少信息

核心公式：
```
Attention(Q, K, V) = softmax(QK^T / √d_k) V
```
- **Q (Query)**：我在找什么
- **K (Key)**：我能提供什么
- **V (Value)**：我实际的内容

### 多头注意力：专家会诊

只做一次注意力，模型可能只能学会关注一种关系。**多头注意力**让模型同时雇请多个"专家"，每个专家从不同角度审视句子：

- 第一个头关注语法结构
- 第二个头关注指代关系
- 第三个头捕捉语义相似性

最后把所有专家的意见拼起来。

### 位置编码：Transformer 不知道谁先谁后

自注意力算的是"谁和谁相关"，但不关心"谁在前面"。

"agent learns" 和 "learns agent" 在它看来是一样的——这显然不合理。

**位置编码**用正弦/余弦函数为每个位置注入独特的位置信息，让模型知道词的顺序。

### 残差连接 + 层归一化

这两兄弟是 Transformer 的稳定器：
- **残差连接**：让梯度能直接跳过层传播，解决深层网络训练难题
- **层归一化**：稳定每层的输入分布，加速收敛

## Decoder-Only：GPT 为什么成功

早期的 Transformer 是 Encoder-Decoder 架构——编码器理解输入，解码器生成输出。翻译任务常用这种结构。

但 OpenAI 发现：**语言模型的核心任务就是预测下一个词**，不需要编码器。

于是有了 **Decoder-Only** 架构，也就是 GPT 系列。

### 自回归生成

```
输入: "今天天气"
模型预测: "好"
输入: "今天天气好"
模型预测: "。"
...
```

每个新生成的词会成为下一步的输入，不断循环。

### 掩码注意力：不能偷看答案

解码时，第 5 个词不能看到第 6、7、8... 个词。

**掩码自注意力**在 Softmax 前，把未来位置的分数置为负无穷，这样就"看不见"了。

### 为什么 Decoder-Only 能成功

- **训练目标单一**：就一个任务——预测下一个词，海量文本直接喂
- **结构简洁**：比 Encoder-Decoder 少一半组件，更容易 Scale Up
- **天生适合生成**：自回归模式和对话、写作、代码生成完美契合

今天的 GPT-4、Llama、Claude，都是这条路上的产物。

## 提示工程的门道

### Temperature：保守还是发散

```
Temperature = 0.1  →  几乎总是选最高概率的词，确定但机械
Temperature = 1.0  →  按原始概率分布采样
Temperature = 1.5  →  更可能选低概率词，更随机更创意
```

**面试话术**：
> "低温度适合代码生成这类需要精确输出的任务，高温度适合头脑风暴或创意写作。"

### Top-k 和 Top-p：候选集的筛选

- **Top-k**：从概率最高的 k 个词里选
- **Top-p (Nucleus)**：从累积概率达到 p 的最小集合里选

Top-p 更智能——概率分布平坦时多选几个，尖锐时少选几个。

### 零样本 vs 少样本

- **零样本**：直接下指令，"把这句话翻译成英文"
- **少样本**：先给例子，让模型"照着做"

少样本提示能显著提升模型在复杂任务上的表现。

## 分词：怎么把文字变成数字

计算机只能处理数字。所以要把文本切成一个个"词元 (Token)"，再转换成 ID。

### 为什么不能简单按词切

- 词表太大（几十万上百万）
- 未登录词 (OOV) 处理不了
- 语义相似的词形变化被割裂

### 子词分词：平衡之道

**BPE (Byte-Pair Encoding)** 是主流方案：

1. 初始词表 = 所有字符
2. 找最常相邻的两字符，合并成新字符
3. 重复，直到词表够大

"agent" 这种常见词整个保留，"unhappiness" 这种长词切成 "un" + "happ" + "iness"——用已知的子词组合未知词。

GPT 用 BPE，BERT 用 WordPiece（合并标准是"最大化语言模型概率"）。

## 面试高频问题

**Q: Transformer 为什么比 RNN 快？**
> RNN 必须顺序计算（第 t 步等第 t-1 步），Transformer 可以并行计算所有位置的自注意力。

**Q: 为什么需要位置编码？**
> 自注意力本身不感知位置信息。"agent learns" 和 "learns agent" 在模型看来完全等价。

**Q: Decoder-only 和 Encoder-Decoder 怎么选？**
> 生成任务选 Decoder-only（对话、写作）；需要理解后生成选 Encoder-Decoder（翻译、摘要）。

**Q: Temperature 0 是什么意思？**
> 退化为贪心采样——永远选概率最高的词，完全确定性。

**Q: 子词分词解决什么问题？**
> OOV（未登录词）和词表膨胀，同时保留语义粒度。

---

## 一句话总结

大语言模型的本质，是一个基于**概率预测**的**序列生成器**。从 N-gram 到 Transformer，进化的核心就两条：**让模型能看到更多上下文**（注意力机制），**让模型能并行处理更多数据**（抛弃循环结构）。
