# 低代码智能体平台

> 适用场景：技术选型、平台对比、面试

---

## 核心价值

| 价值 | 说明 |
|------|------|
| 降低门槛 | 非技术人员也能构建智能体 |
| 快速验证 | 数小时完成原本数天的开发 |
| 可视化调试 | 图形化查看执行轨迹 |
| 最佳实践 | 内置模板和标准化组件 |

---

## 平台对比

| 平台 | 定位 | 核心优势 | 适用场景 |
|------|------|----------|----------|
| **Coze** | 零代码 Agent 构建 | 插件生态丰富、一键多平台发布 | 快速验证、C端应用 |
| **Dify** | 开源 LLM 应用开发平台 | 企业级安全、MCP 支持、可自部署 | 生产环境、企业应用 |
| **n8n** | 工作流自动化 + AI | 强在"连接"能力，通用自动化 | 业务集成、复杂工作流 |

---

## Coze

**核心特点**：
- 可视化工作流编排，拖拽式构建
- 插件市场丰富（RSS、GitHub、arXiv 等）
- 一键发布到抖音、飞书、豆包等平台

**局限**：
- 不支持 MCP（致命缺点）
- 复杂工作流仍需一定编程基础

**面试话术**：
> "Coze 的优势在于生态和发布效率，适合快速验证想法。但不支持 MCP 是硬伤，企业级场景受限。"

---

## Dify

**核心特点**：
- 开源，支持私有化部署
- 内置 RAG Pipeline、知识库管理
- 支持 9000+ 工具和 MCP
- 提供审计日志、RBAC 等企业级安全能力

**局限**：
- 学习曲线较陡
- Python 实现，高并发场景有性能瓶颈
- API 格式不兼容 OpenAI

**面试话术**：
> "Dify 是最接近生产级需求的平台，可自部署保证数据安全，9000+ 工具生态完善。但性能是短板，大流量场景需要优化。"

---

## n8n

**核心特点**：
- 本质是工作流自动化工具，LLM 是其中一环
- 数百个预置节点，连接一切
- 支持自定义代码节点

**局限**：
- 非专为 LLM 设计，AI 能力是后来集成的
- 学习曲线最陡

**面试话术**：
> "n8n 的强项是连接能力，适合需要深度集成现有系统的场景。如果你需要的是一个'AI 增强的自动化流程'，选 n8n；如果是专门的 Agent 应用，选 Coze 或 Dify。"

---

## 选型决策

```
需要快速验证 / 非技术团队 → Coze
需要生产部署 / 企业级 → Dify
需要深度集成现有系统 → n8n
```

---

## 一句话总结

低代码平台是"更高层次的抽象"——让开发者从底层工程实现中解放，专注于业务逻辑。但它们不是银弹，复杂定制场景仍需要代码能力。
