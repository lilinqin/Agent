# Agent 本质

> 适用：技术面试、概念阐述

---

## 一句话定义

**Agent = LLM + 工具调用 + 循环反馈**

它是能在环境中持续运作的自主系统：感知→决策→行动→观察，不断迭代直到目标达成。

---

## 核心循环：ReAct

```
Thought: 分析情况
Action: 调用工具
Observation: 结果反馈
→ 下一轮 Thought...
```

**关键**：把非结构化的语言推理，和结构化的工具调用，用明确协议桥接起来。

---

## Agent vs Workflow

| | Workflow | Agent |
|---|---|---|
| 执行方式 | 确定性，预设流程 | 概率性，自主规划 |
| 适用场景 | 规则清晰、边界已知 | 任务模糊、情况多变 |
| 灵活性 | 低 | 高 |

**最优解是混合架构**：LangGraph 用状态图控主干，动态节点嵌入 Agent。

---

## 能力上限取决于

1. **LLM 推理质量**
2. **工具覆盖范围**
3. **上下文窗口管理**

---

## 工程挑战

- **上下文膨胀**：每轮循环加 Thought/Action/Observation，几轮后 token 就满了
- **决策漂移**：早期关键信息被稀释，Agent 忘了最初目标
- **调试困难**：没有堆栈，只有对话记录

**成熟系统需要**：完善 tracing + Human-in-the-loop

---

## 面试高频

**Q: Agent 和普通 LLM 调用的区别？**
> "普通 LLM 是一次性调用，Agent 是循环迭代。核心是有反馈机制——Action 的结果（Observation）会反馈给 LLM 做下一轮决策。"

**Q: 什么时候用 Workflow 不用 Agent？**
> "规则固定、边界清晰、可审计性要求高的场景（如退款审批）。Agent 适合情况多变、分支写不完的场景（如合同风险分析）。"
